# 文本评估系统使用指南

## 什么时候用

在以下时机对生成的文本运行评估：

1. **每次生成完整章节后**：写完一章就检查一次，不要等全文写完
2. **用户要求检查时**：用户说"帮我检查一下""看看有什么问题"
3. **定稿前最终检查**：提交/投稿前做一次全面评估

## 怎么用

### 命令行调用

```bash
# 评估一个文件
python scripts/review.py paper.md

# 只看错误级别
python scripts/review.py paper.md --severity error

# JSON输出（方便程序处理）
python scripts/review.py paper.md --format json
```

### Agent 内部调用

```python
from scripts.review import review

text = "论文文本内容..."
issues = review(text)

for issue in issues:
    print(f"[{issue.severity.value}] {issue.rule}: {issue.message}")
```

## 检查维度

| 编号 | 维度 | 检查内容 | 严重级别 |
|------|------|---------|---------|
| R1-01 | 可信度 | 「有学者指出」等模糊引用 | 错误 |
| R1-02 | 可信度 | [待补充] 占位符（提醒补充） | 提示 |
| R1-03 | 可信度 | 引用年份超过当前年份 | 错误 |
| R3-01 | 可信度 | 过度断言（毫无疑问、众所周知等） | 警告 |
| T-01 | 术语 | 同一概念使用多个中文译名 | 警告 |
| F-01 | 格式 | 空脚注 | 错误 |
| F-02 | 格式 | 参考文献缺少文献类型标识 | 警告 |
| F-03 | 格式 | 中文语境使用英文标点 | 警告 |
| F-04 | 格式 | 标题编号体系混用 | 警告 |
| S-01 | 语体 | 口语化表述 | 警告 |
| S-02 | 语体 | 自称用法不统一 | 警告 |
| L-01 | 论证逻辑 | 段落内连续多句缺少论证连接（只论不证） | 警告 |
| L-02 | 论证逻辑 | 连续多段以并列词开头（只证不论） | 警告 |
| L-03 | 论证逻辑 | 引文后缺少分析，直接跳到新话题 | 警告 |
| L-04 | 论证逻辑 | 章节开头缺少与上一章的逻辑衔接 | 提示 |
| L-05 | 论证深度 | 因果前提含价值判断但未论证（"由于X并非Y"） | 警告 |
| L-06 | 论证深度 | 长句中叙述成分过多，缺少分析性表述 | 提示 |
| L-07 | 论证深度 | 总结句堆砌了前文未充分展开的并列概念 | 警告 |
| L-08 | 论证深度 | 强度词（显然、极其）附近缺少充分论据 | 警告 |
| ST-01 | 结构 | 缺少摘要/关键词/参考文献 | 提示 |
| ST-02 | 结构 | 引言缺少论点句 | 警告 |

## 评估系统的边界

这个评估系统**能做的**：
- 用正则匹配发现确定性的格式和用词问题
- 比对术语表检查译名一致性
- 检查结构要素是否完整

这个评估系统**不能做的**（需要人工或模型辅助判断）：
- 判断论点是否有说服力
- 判断论证逻辑是否连贯
- 判断文献引用是否准确（需要对照原文）
- 判断理论运用是否恰当

对于后者，应在 SKILL.md 的方法论指导下，由 agent 和用户共同判断。
