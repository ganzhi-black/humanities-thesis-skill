# 核心文献拆解流程

筛选出核心文献后，如何系统拆解以实现最大化利用，同时不对文献内容产生主观臆断。

---

## 前置要求：确保读到了原文

拆解的前提是读到文献的**原文内容**，而不是凭标题和摘要推测。

**文件读取流程**：
1. 用户上传 PDF / DOCX / 其他格式文件后，使用 agent 的文件读取能力解析全文
2. 如果是 PDF，优先尝试直接提取文本；如果是扫描件（图片型PDF），使用 OCR 工具处理
3. 如果文件无法解析，告知用户并建议：将文献关键章节手动粘贴到对话中
4. **绝对不要在没有读到原文的情况下"拆解"文献**——那不是拆解，是编造

---

## 拆解的五个层次

对每篇核心文献，按以下五个层次逐层拆解。每个层次都只从文献原文中提取信息，不添加主观推断。

### 第一层：元信息提取

从文献中提取以下基本信息，形成一张卡片：

```
标题：
作者：
年份：
期刊/出处：
核心论点（用一句话概括）：
研究对象：
理论框架：
方法论：
关键词：
```

**注意**：核心论点必须从论文自身的论点句中提取（通常在摘要或引言末尾），不要用自己的话重新概括——重新概括容易无意中改变论点的方向。

### 第二层：论证结构拆解

梳理文献的**论证逻辑链条**：

1. **问题**：作者提出了什么问题？这个问题的学术背景是什么？
2. **立场**：作者对已有研究持什么态度？同意谁？反对谁？
3. **论证路径**：从问题到结论，作者经过了哪几步？每步的核心论据是什么？
4. **结论**：最终论点是什么？作者自己承认了哪些局限？

拆解时按章节顺序走，逐章记录该章的核心任务和推进逻辑。不要跳过章节，不要合并段落。

### 第三层：证据与引文标记

标记文献中**可以被你的论文直接使用的内容**：

| 标记类型 | 说明 | 提取方式 |
|---------|------|---------|
| 可引用论点 | 作者的原创论断 | 原文摘录 + 页码 |
| 可引用数据/事实 | 作者引用的历史事实、统计等 | 原文摘录 + 页码，同时标注数据的**原始来源**（作者也可能是转引的） |
| 可引用方法 | 作者的分析方法值得借鉴 | 记录方法描述 + 页码 |
| 需要回应的观点 | 你不同意或需要补充的论断 | 原文摘录 + 你的初步回应方向 |

**关键规则**：所有标记必须附带页码。没有页码的引用在学术论文中不可接受。

### 第四层：与你论文的关系定位

在拆解完前三层之后（不是之前），才判断这篇文献和你的论文之间的关系：

| 关系类型 | 判断标准 | 在论文中如何使用 |
|---------|---------|----------------|
| 理论基础 | 你的分析框架建立在这篇文献之上 | 引言或理论框架部分引用 |
| 对话对象 | 你同意其部分观点但要推进/修正 | 文献综述中作为对话方 |
| 反面论据 | 你要明确反对其论点 | 正文中设置张力 |
| 方法借鉴 | 你的分析方法参考了它 | 方法论部分引用 |
| 背景材料 | 提供历史或语境信息 | 在需要语境的地方引用 |

### 第五层：交叉比对

如果拆解了多篇核心文献，做一次交叉比对：

- 这几篇文献之间存在什么共识？
- 存在什么分歧？分歧的焦点在哪里？
- 它们共同忽略了什么？（这往往指向你的论文可以填补的空白）
- 它们各自的证据是否能相互验证或相互矛盾？

## 防止臆断的检查清单

拆解完成后，用以下清单自查：

| 检查项 | 通过标准 |
|--------|---------|
| 每个论点归纳都能指向文献原文的具体位置 | 有页码/段落编号 |
| 没有把自己的推测混入对文献的描述中 | 所有评价性语言都标注了"本文认为"而非直接断言 |
| 区分了"作者说了什么"和"这意味着什么" | 描述与分析分开 |
| 如果文献的某部分没读到原文，标注了"未见原文" | 不凭摘要推测正文内容 |
| 引文的上下文没有被截断导致意思改变 | 回原文检查引文前后文 |

---

## 常见拆解错误

| 错误 | 表现 | 后果 |
|------|------|------|
| 只读摘要就"拆解" | 把摘要的概括当作文献的全部论点 | 遗漏论证的细节和张力 |
| 带着预设去拆解 | 只提取支持自己观点的内容 | 选择性引用，学术不诚实 |
| 混淆作者观点和转引观点 | 把作者引用的别人的话当作作者自己的话 | 错误归因 |
| 拆解变成了复述 | 逐段翻译文献内容 | 没有分析价值 |
| 忽略作者的自我限定 | 作者说了"可能""在某种程度上"，转述时去掉了 | 歪曲了论点的强度 |
